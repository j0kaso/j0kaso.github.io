When I started my PhD, now almost 3 years ago, the standard of sharing looked like this: Depending on where MD studies were presented, trajectory output was converted into text, figures and at the best videos. Eager to understand the biological processes, in my case G-Protein Coupled Receptors and to run simulations on them by myself, I looked up those studies and asked myself what actually is happening there, as figures are just static and videos only show a certain in-transparent perspective. This doesn't tell us the whole story of these complex dynamics! So I went to my supervisor and asked him, if there is something out which is actually more interactive, having in mind something like this, where I can easily change the perspective in an instance and being able to zoom & focus on regions, I'm interested in.

As MD belongs to the data-intensive sciences, sharing is hindered by large file sizes and different file formats. Often, trajectories are unprocessed. Additionally, visualizing MD trajectories demand specialized, standalone software, setups and processed trajectories. So in the end, expert knowledge. This heavily limits the audience to which MD is available and besides the technical aspects also the degree to which MDs can be shared visually. Luckily, I found that solutions for most of the problems are already available. For instance - the best and easiest way to share is via the web.

Until recently, web browsers could not display 3D content without additional plug-ins like Java Applets or Flash. With the rise of the web as a platform for applications, more capabilities have been added to the web browsers, including HTML5 and the web browser programming language JavaScript. Together with its access to dedicated graphics hardware through WebGL API to calculate on the GPU directly, rendering performance and quality has notably improved. By now, WebGL and HTML5 is standard in every modern web browser. This improvements allowed my former colleague Alex Rose the development of the NGL Viewer in our lab.

This example show us that by utilizing the NGL viewer, it is easily possible to visualize several million atoms, so large file sizes and huge amount of data as generated by MD are in principle no problem, especially, if you only transfer the information needed to visualize the input by the user. Still, at this moment, NGL could not efficiently show MDs.

So, the MDsrv is a program for interactive and remote exploration of trajectories by employing a client-server architecture or in short a tool to stream trajectories over the web. And within this Browser-based presentation, you can already see such a toy example, which is by the way a simulation of the A2A receptor where Cholesterol is entering into the ligand binding pocket and was run by my former colleague Ramon Guixa-Gonzales at Jana Selents lab. If you want, you can now open this link to explore an example of what our collaboration partners, reviewers or my colleges get. I will go with you through some options of the MDsrv.
  • prepared session
  • predefined trajectory options, repres., functions
  • GUI, easily change settings
  • trajectory settings like run-step time, interpolation, centering
  • selection, what you want to visualize
  • export high-resolution images & download PDB-formatted files
  • that's what your collaboration partners get - of course hidden & password protected
  • how you as experts can benefit
  • after installation: conda / pip
  • change to project dir, execute mdsrv
  • load structure & trajectory file
  • single parts automatically concatenated
  • raw traj → remoce PBC, superpose with first frame/structure, center to input
  • esp. powerful: connected via fusesh to cluster
  • make available to others like 1 example, install it in your apache server
  • how helpful for students & non-experts: exemplary wiki
  • think about traj showing activating mechanisms → instant intuitive understanding via vis
This example is a prepared session, where some trajectory options, representations or functions are already predefined. As you're within the GUI, you can easily change those settings again, like here, where you can change the trajectory settings like run-step time or interpolations. Here, you can change the selection, meaning what you want to visualize. You can also make high-resolution figures and download PDB-formatted files. Here you got a glimpse on what our collaboration partners get,of course hidden & password protected. I want to show you now also, how you as expert can benefit from the mdsrv. After installation via conda or pip, you can just change to your project directory, execute the mdsrv and load a structure & trajectory file. Single parts will be automatically concatenated. As you can see, this is a raw trajectory but we can easily remove the PBC, superpose it with the first frame or structure and center it according to our input. This is especially powerful if you connect for example via fusesh to your cluster. To make this available to others like the first example, you have to install it in your apache server. How helpful this is also for students and non-experts can be seen here in this exemplary wiki. You can easily think about adding a trajectory showing activating mechanisms allowing instant intuitive understanding via visualization.
